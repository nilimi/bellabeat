Prepare

Data

The data was obtained from https://www.kaggle.com/arashnic/fitbit with the following description:

This dataset generated by respondents to a distributed survey via Amazon Mechanical Turk between 03.12.2016-05.12.2016. Thirty eligible Fitbit users consented to the submission of personal tracker data, including minute-level output for physical activity, heart rate, and sleep monitoring. Individual reports can be parsed by export session ID (column A) or timestamp (column B). Variation between output represents use of different types of Fitbit trackers and individual tracking behaviors / preferences.

The data is reported to be of 30 self-selected users (33 distinct users during analysis). This small sample size may not reflect the population of Bellabeat users. Self-selected users are not random. The data may be biased. For instance, users willing to participate in the study may be more inclined to be more active than an average user. Additionally, Fitbit users may differ from Bellabeat users.

Process

Analyze

During exploration, I noticed that there were a different number of users for three of the data tables. This difference is where I focused the analysis.

I created a new data frame containing the name of each table and the distinct number of users.

table_name <- c("daily_activity", "sleep_day", "daily_steps", "heart_rate_seconds", "hourly_calories", "hourly_intensities", "hourly_steps", "minute_calories_narrow", "minute_intensities_narrow", "minute_mets_narrow", "minute_steps_narrow", "weight_log")

number_of_users <- c(n_distinct(daily_activity$Id), n_distinct(sleep_day$Id), n_distinct(daily_steps$Id), n_distinct(heart_rate_seconds$Id), n_distinct(hourly_calories$Id), n_distinct(hourly_intensities$Id), n_distinct(hourly_steps$Id), n_distinct(minute_calories_narrow$Id), n_distinct(minute_intensities_narrow$Id), n_distinct(minute_mets_narrow$Id),  n_distinct(minute_steps_narrow$Id), n_distinct(weight_log$Id))

number_of_users_per_table <- data.frame(table_name, number_of_users)

number_of_users_per_table <- number_of_users_per_table %>% arrange(desc(number_of_users))

number_of_users_per_table$color <- ifelse(number_of_users_per_table$number_of_users < 33, "red", "gray")

number_of_users_per_table$table_name <- factor(number_of_users_per_table$table_name, levels = number_of_users_per_table$table_name)

Share

I created a bar char to visualize the difference in the number of users in the different tables.

ggplot(number_of_users_per_table, aes(x = table_name, y = number_of_users, fill = color)) +
  geom_bar(stat = "identity") +
  labs(title = "Number of Users in Each Table", x = "Tables", y = "Number of Users") +
  theme(axis.text.x = element_text(angle = 45, hjust = 1)) +
  guides(fill = "none")

Act

The bar chart illustrates that while 33 users participated in nine of the different types of records created, less user participated in three of the tables: sleep, heart rate and weight.

My recommendation would be to determine why users would not or could not submit the sleep, heart rate and weight data. Is this something the users would like to track but are running into difficulties? If the Fitbit device or app lacks in this area, by either device malfunction or user interface difficulties, Bellabeat has an opportunity to improve upon these areas and possibly convert Fitbit users to Bellabeat.
